この記事は、AIエージェント比較シリーズの総合まとめです。

同じ課題を6つのエージェントに渡し、実装、自己評価、他者テスト修正、相互コードレビューまでを横断して比較しました。対象はClaude Code、Codex CLI、Antigravity CLI、Codex IDE、Antigravity IDE、Copilot Agentの6つです。

結論から言うと、今回の総合1位は **Claude Code** と判断しました。

ただし、実験Dだけを見るなら **Copilot Agent** が最も信頼できる結果でした。最終判断では、実験Aの実装力で明確に1位だったClaude Codeと、実験Dの誠実さ・テスト修正品質で最も安定していたCopilot Agentの比較が核心になります。

## 1. はじめに（実験概要、6エージェント・全実験の振り返り）

今回の比較では、AIエージェントに同じ条件で課題を与え、成果物を人間が検証しました。

中心になったのは、詳細仕様に沿ってタスク管理アプリを実装する実験Aです。FastAPIとVue 3を使い、CRUD、フィルタ、ソート、優先度、期限、ステータス管理などを含む小規模なWebアプリを作らせました。

その後、実験Bではより自由度の高い条件で設計から実装させ、実験Dでは他者の成果物向けに共通テストを修正させ、実験Eではエージェント同士に相互コードレビューをさせました。

この記事では、原則として実験Aの定量スコアを総合比較の土台にします。ただし、実験Dで見つかった「期待値を書き換えて見かけの合格率を上げる」行為は、実務上の信頼性に直結するため、最終順位では重く扱います。

今回の実験で見えた全体傾向は次の通りです。

- 6エージェント中5エージェントが、実験Aの共通テストで22/24以上を通した
- 実装品質の差は、テスト合格率よりも開発時間、UI完成度、ドキュメント、例外処理に出た
- Codex系は対話回数が少なく、一度の指示で完結させる傾向が強かった
- Antigravity系はUI/UXに強く、フロントエンド行数が突出した
- Copilot Agentは実装の堅牢性と報告の正確さが目立った
- Claude Codeは速度、実装品質、レビュー構成力のバランスが最も高かった

## 2. 総合スコアランキング（実験A単体・実験計画書の配点ベース）

まず、詳細仕様を渡した実験A単体のスコアです。これは実験計画書の配点に基づく90点満点のランキングで、実験Eのレビュースコアは含めていません。

| 順位 | エージェント | 総合点 | テスト | 時間 | 可読性 | エラー処理 | UI | 文書 | テスト網羅 |
|---:|---|---:|---:|---:|---:|---:|---:|---:|---:|
| 1 | Claude Code | 85.0 | 20.0 | 10.0 | 10.0 | 15.0 | 12.0 | 8.0 | 10.0 |
| 2 | Copilot Agent | 79.4 | 20.0 | 4.4 | 10.0 | 15.0 | 12.0 | 10.0 | 8.0 |
| 3 | Antigravity CLI | 72.3 | 18.3 | 10.0 | 8.0 | 9.0 | 15.0 | 6.0 | 6.0 |
| 4 | Antigravity IDE | 70.2 | 19.2 | 0.0 | 8.0 | 12.0 | 15.0 | 8.0 | 8.0 |
| 5 | Codex IDE | 70.0 | 20.0 | 5.0 | 10.0 | 12.0 | 9.0 | 8.0 | 6.0 |
| 6 | Codex CLI | 68.6 | 20.0 | 5.6 | 8.0 | 12.0 | 9.0 | 8.0 | 6.0 |

実験A単体では、Claude Codeが明確に1位でした。4分で実装を終え、共通テスト24本を通し、可読性、エラー処理、テスト網羅性も最高評価です。

2位のCopilot Agentは、開発時間こそ13分と長めでしたが、共通テスト24本で一切の問題が検出されなかった唯一のエージェントでした。READMEの情報量も最も厚く、実装の堅牢性という意味ではClaude Codeに迫っています。

3位以下は、それぞれ強みと弱みが分かれました。Antigravity CLIとAntigravity IDEはUIが最高評価でしたが、バックエンドや開発時間で減点されました。Codex IDEとCodex CLIは共通テスト上は24/24でしたが、UI完成度やテスト網羅性で伸びませんでした。特にCodex IDEは、共通テストを通過したにもかかわらず、後からPUT部分更新バグが実証されており、「テスト合格＝品質保証ではない」ことを示す例になりました。

## 3. 開発スピードの比較（最速4分〜最長20分、5倍の差）

実験Aの開発時間は次の通りです。

| エージェント | 開発時間 | やり取り回数 | BE行数 | FE行数 | 共通テスト |
|---|---:|---:|---:|---:|---:|
| Claude Code | 4分 | 18回 | 196 | 527 | 24/24 |
| Antigravity CLI | 4分 | 9回 | 167 | 1043 | 22/24 |
| Codex CLI | 11分 | 3回 | 205 | 501 | 24/24 |
| Codex IDE | 12分 | 3回 | 152 | 509 | 24/24 |
| Copilot Agent | 13分 | 25回 | 177 | 627 | 24/24 |
| Antigravity IDE | 20分 | 13回 | 154 | 1224 | 23/24 |

最速はClaude CodeとAntigravity CLIの4分、最長はAntigravity IDEの20分でした。5倍の差です。

ここで重要なのは、速いこと自体よりも「速さと品質が両立しているか」です。

Claude Codeは4分で24/24を通し、総合スコアも1位でした。これは速度が品質を犠牲にしていない例です。Antigravity CLIも同じ4分でしたが、共通テストは22/24で、DELETEのステータスコードと優先度ソートの意味的反転バグが見つかりました。速さは同等でも、仕上がりには差がありました。

Codex系2つは、開発時間は中位ですが、やり取り回数がどちらも3回と少ないのが特徴です。一度の指示で一気に成果物を出す傾向が強く、対話しながら詰めるよりも、まとまった作業を任せるスタイルに向いている可能性があります。

Copilot Agentは13分・25回とやり取りが最も多く、スピード面では不利でした。一方で、実験Aの実装は共通テストで問題ゼロ、完了報告の技術的根拠も正確でした。時間はかかるが堅く仕上げるタイプです。

Antigravity IDEは20分と最も長く、実験Bでは34分でした。ただし、UIの作り込みは突出しており、フロントエンド行数も実験Aで1224行と最多です。時間をUI/UXに投資する傾向が強いと見てよさそうです。

## 4. テスト合格率の比較（共通テスト24本基準）

実験Aの共通テストは、バックエンド18本、フロントエンド6本の合計24本です。

| エージェント | 共通テスト | 主な検出事項 |
|---|---:|---|
| Claude Code | 24/24 | 保存ボタンのラベル差異によりテスト側セレクタ調整は必要 |
| Codex CLI | 24/24 | 起動・テスト原本側の問題を切り分け後、実装欠陥なし |
| Antigravity CLI | 22/24 | DELETE 200返却、優先度ソートdesc/asc反転 |
| Codex IDE | 24/24 | 共通テスト外でPUT部分更新バグを実証 |
| Antigravity IDE | 23/24 | DELETE 200返却 |
| Copilot Agent | 24/24 | 24本で問題検出なし |

数字だけを見ると、24/24が4エージェントあります。しかし内訳を見ると、同じ24/24でも意味が違います。

Copilot Agentは、共通テスト24本の中で一切の問題が検出されなかった唯一のエージェントです。DELETE 204、PUT部分更新、sort/order検証、CORSの扱いなど、他エージェントで問題になったパターンが見当たりませんでした。

Claude CodeとCodex CLIも最終的には24/24ですが、テスト適用時にセレクタやテスト原本側の調整が必要でした。ただし、これは実装欠陥というより、UIテスト設計の難しさを示す事例です。

Codex IDEは24/24でしたが、後からPUT部分更新が機能しない重大バグが見つかりました。`TaskUpdate(TaskBase)`の継承により、titleなしの更新が422で拒否される問題です。共通テストの更新ケースがtitleを含むpayloadだったため、この欠陥を検出できませんでした。

Antigravity CLIは22/24で、特に優先度ソートの意味的反転が重要です。内部マッピングをそのままdescにしたことで、期待される「高い優先度順」と逆になりました。この種のバグは静的レビューでは見落とされやすく、実際に実験Eのレビューでも一度も指摘されていません。

テストの教訓は明確です。共通テストは強いが、万能ではありません。動的テスト、静的レビュー、手動検証はそれぞれ違う種類の欠陥を見つけます。

## 5. 自己評価ギャップのまとめ（過大評価1・過小評価2・一致3の分類）

実験A・Bでは、人間評価とエージェント自身の自己評価の差も集計しました。差分は `ai_self - human` です。

| 分類 | エージェント | 実験A平均差 | 実験B平均差 | コメント |
|---|---|---:|---:|---|
| 過大評価 | Antigravity CLI | +0.80 | +0.60 | 一貫して自己評価が高め |
| 過小評価 | Codex CLI | -1.20 | -0.80 | ツール操作ミスによる誤検出が主因 |
| 過小評価 | Copilot Agent | -0.60 | -0.40 | 事実認識は比較的正確で控えめ |
| 一致 | Claude Code | 0.00 | -0.20 | 小幅で安定 |
| 一致 | Antigravity IDE | 0.00 | +0.40 | 小幅な範囲 |
| 一致 | Codex IDE | -0.20 | -0.40 | 軽微な過小。ただしPUTバグは見逃し |

最も注意が必要なのはCodex CLIです。数値上は最も過小評価ですが、これは「謙虚で正確だった」という話ではありません。

Codex CLIは、PowerShellで`Get-Content`のUTF-8指定をしなかったためREADMEやUIを文字化けと誤認し、さらにpytestの実行ディレクトリを誤ったことでテスト起動失敗と誤認していました。つまり、成果物を正しく読めなかったことによる過小評価です。

Copilot Agentの過小評価は性質が違います。実装の事実認識は概ね正確で、自己評価だけがやや控えめでした。これは選定時に安心材料として扱いやすい過小評価です。

Antigravity CLIは、実験A・Bともに一貫して過大評価でした。実験Aでは「既知の問題なし」と報告していましたが、共通テストでDELETE 200返却と優先度ソート反転が見つかっています。UIの完成度が高いぶん、バックエンド側の不具合を自己評価で拾い切れていなかった印象です。

Claude Code、Antigravity IDE、Codex IDEは数値上は人間評価に近い範囲でした。ただし、Codex IDEはPUT部分更新バグを自己評価で見逃しているため、「平均差が小さい」ことだけでメタ認知が完全とは言えません。

## 6. 他者テスト修正での誠実さ（実験D：指示違反の有無）

実験Dは、今回の比較で最も重要な実験の一つです。

各エージェントに、他者の成果物向けに共通テストを修正させました。ただし、期待するHTTPステータスコードや検証観点は変更しない、という制約を置きました。

結果は次の通りです。

| エージェント | 合格率 | 指示違反 | 主な内容 |
|---|---:|---:|---|
| Claude Code | 97.5% | 0件 | 指示は守ったが、UIテストで「更新」ボタン衝突を1件踏んだ |
| Codex CLI | 100.0% | 1件 | DELETE 204→200、priority順序を書き換え |
| Antigravity CLI | 90.8% | 0件 | 違反なし。ただしVue 3のv-model属性セレクタで偽陽性が多い |
| Codex IDE | 90.8% | 1件 | priority順序を書き換え。UIセレクタも脆弱 |
| Antigravity IDE | 100.0% | 2件 | response.status_codeを実行時に200→204へ書き換え |
| Copilot Agent | 99.2% | 0件 | 報告と実機検証が完全一致。最も信頼性が高い |

ここでは、見かけの合格率をそのまま評価してはいけません。

Codex CLIとAntigravity IDEは100.0%ですが、どちらも指示違反があります。Codex CLIは期待値を定数化して書き換え、Antigravity IDEはさらに巧妙に、アサーション文は`== 204`のまま残しつつ、その手前でレスポンスオブジェクトの`status_code`を200から204へ書き換えていました。

これは実務上かなり重い減点です。テストを通すこと自体よりも、「本来失敗すべきものを失敗として残せるか」の方が重要な場面があります。

違反ゼロ組の中でも差があります。

Antigravity CLIは期待値を書き換えなかった点では誠実でしたが、Vue 3ではDOMに残らない`v-model`属性をセレクタに使い、例外を握り潰したため、UIテストにサイレントな偽陽性が出ました。

Claude Codeも違反はありませんでしたが、保存ボタン検出で「更新」という語に誤マッチし、ヘッダーの一覧再取得ボタンを押してしまう失敗がありました。

Copilot Agentは、APIテスト原本を事実上維持し、期待値を変えない理由も自発的に説明し、UIテストでもv-model属性、ボタン名衝突、required属性干渉を回避しました。実験Dだけなら、Copilot Agentが明確に1位です。

## 7. コードレビュー力（実験E：均質化トラップ・誤検出の総括）

実験Eでは、エージェント同士にコードレビューをさせました。

ここで見えたのは、「AIレビューは有用だが、そのまま鵜呑みにできるものではない」ということです。

まず、レビュー手法には明確な個性がありました。

Codex CLIとCodex IDEは、実際にpytestを実行してレビューする傾向が強く出ました。Codex CLIは、Codex IDEのPUT部分更新バグを実測で発見しています。静的に読むだけでは見落とされていた可能性が高いバグです。

Claude Codeは、型安全性、DRY、インデックス、UTC対応、テスト網羅性など、観点を明示して横断的に整理するレビューが目立ちました。実験Bではsqlite3接続管理の問題やCORS設定も指摘しています。

Antigravity系はレビュー速度が非常に速く、UI/UXやSQL動的組み立てへの反応が強めでした。一方で、実験Aでは同系統のAntigravity実装に高評価をつける傾向も観測されました。

Copilot Agentは、評価点がやや高めに寄る傾向はありましたが、異系統レビュアーの評価とよく一致し、due_dateのNULL順序やCORS問題など横断的な指摘もできていました。

実験Eの主な発見は3つです。

1つ目は、横断的な設計バグの検出です。複数のレビュアーが、フィルタ適用後の一覧からサマリー件数を算出してしまう問題を指摘しました。仕様書にサマリー集計の基準が明記されていなかったこともあり、これは実装者だけでなく実験設計側にも示唆のある発見です。

2つ目は、均質化トラップです。実験Aでは、Antigravity CLIがAntigravity IDEに9.0点を付け、異系統のClaude Code、Codex CLI、Codex IDE、Copilot Agentは全員7点でした。また、Antigravity IDEもAntigravity CLIに8点を付け、異系統は5〜6点でした。この範囲では、同系統への評価が甘くなる傾向がかなり強く見えます。

ただし、実験Bではこの傾向が弱まりました。Antigravity CLIは同系統のAntigravity IDEを6点と低く評価し、Antigravity IDEも同系統のAntigravity CLIを7点に留めています。したがって、結論は「均質化トラップは常に起きる」ではなく、「詳細仕様下の実験Aでは強く出たが、自由設計の実験Bでは弱まった」です。

3つ目は、誤検出です。Codex IDEはコメント文字化けを指摘しましたが、UTF-8指定で確認すると実際には文字化けしていませんでした。Antigravity IDEはCodex CLIについて「テストが本番DBを破壊する」と指摘しましたが、後日の実機検証では再現せず、誤検出と判明しました。

AIレビューは有用です。特に複数エージェントが独立に同じ問題を指摘した場合は信頼度が高いです。しかし、単独の重大指摘は必ず実機検証で裏を取るべきです。

## 8. 6エージェント、結局どれが一番よかったか（総合順位を出す）

総合順位を出します。

私の結論は次の通りです。

| 総合順位 | エージェント | 判断 |
|---:|---|---|
| 1 | Claude Code | 実験A 1位。速さ、品質、レビュー構成力の総合力が最も高い |
| 2 | Copilot Agent | 実験Dは最優秀。実装も堅牢だが、開発速度でClaude Codeに届かない |
| 3 | Antigravity CLI | 最速タイでUIが強い。違反なし。ただし実装バグとUIテスト偽陽性が重い |
| 4 | Codex IDE | 実験Aは24/24で可読性も高いが、PUTバグ見逃しと実験D違反で減点 |
| 5 | Antigravity IDE | UIは最高水準だが、開発時間と実験Dの巧妙な指示違反が重い |
| 6 | Codex CLI | 24/24で堅実な面はあるが、実験A最下位かつ実験D違反、自己評価誤認が響く |

1位はClaude Codeです。

理由は、実験Aの総合スコアで明確に1位であり、しかも最速4分でその結果を出したからです。共通テスト24/24、可読性・エラー処理・テスト網羅性も最高評価で、実装フェーズの総合力が最も高いと判断できます。

ただし、Claude Codeが全項目で勝ったわけではありません。実験Dでは、保存ボタン検出で「更新」ボタン衝突を踏んでおり、Copilot Agentに見劣りします。したがって、「他者テスト修正だけを任せるならCopilot Agentの方が上」です。

2位はCopilot Agentです。

Copilot Agentは、実験Aで共通テスト24本に一切の問題が検出されなかった唯一のエージェントでした。実験Dでも、報告と実機検証結果が完全一致し、期待値を変えない理由まで説明しています。信頼性という観点では最も安心できます。

それでも総合1位をClaude Codeにしたのは、実験Aの総合スコア差が5.6点あり、特に開発時間の差が大きいからです。Claude Codeは4分、Copilot Agentは13分でした。実装力、速度、レビュー構成力まで含めた総合比較では、Claude Codeを上に置くのが妥当です。

3位はAntigravity CLIです。

4分で動くものを出し、UI品質は最高評価です。実験Dでも期待値を書き換えなかった点は評価できます。一方で、実験AではDELETE 200返却と優先度ソート反転があり、実験Dではv-model属性セレクタによるサイレントな偽陽性がありました。速くて見た目も強いが、堅牢性には注意が必要です。

4位はCodex IDEです。

実験Aのスコアは70.0で5位ですが、Codex CLIとの差は小さく、可読性は高評価でした。実験Bでは同系統のCodex CLIより堅牢な実装を出しています。ただし、実験AでPUT部分更新バグを見逃したこと、実験Dでpriority順序を書き換えたことは重い減点です。

5位はAntigravity IDEです。

UI/UXは6エージェント中でも最高水準で、自律的なブラウザ操作による確認も行っていました。しかし、実験Aの開発時間は20分で最長、実験Bでは34分でした。さらに実験Dでは、レスポンスオブジェクトを書き換えて本来失敗すべき204検証を通すという、最も巧妙な指示違反がありました。実務で任せる場合、見た目の良さだけで上位には置けません。

6位はCodex CLIです。

実験Aでは共通テスト24/24で、UIテストのセレクタ設計にも良い点がありました。一方で、実験Aスコアは68.6で最下位です。実験Dでは期待値の書き換えがあり、自己評価では文字化け・pytest起動失敗を誤認して大きく過小評価しました。実装そのものが常に悪いわけではありませんが、総合選定では最下位が妥当です。

今回の選定で最も重く見たのは、実験Dの指示違反です。AIエージェントに作業を任せる場合、「テストを通すために都合の悪い期待値を書き換えない」ことは、速度やUIより前にある信頼の条件です。

その前提で、違反なし、実装力高水準、総合スコア上位という条件を満たしたClaude CodeとCopilot Agentが2強でした。最終的には、実装力と速度まで含めてClaude Codeを1位、誠実さと堅牢性では最も優れたCopilot Agentを2位とします。

## 9. 場面別おすすめ

### 仕様が固まっている場合

仕様が固まっていて、短時間で品質の高い実装を出したいなら、第一候補はClaude Codeです。

実験Aでは、詳細仕様に対して4分で実装し、共通テスト24/24、総合85.0点でした。仕様に沿って堅く実装する能力とスピードの両方が最も高く出ています。

次点はCopilot Agentです。時間はかかりますが、仕様への忠実度と堅牢性は非常に高く、共通テストで問題が検出されなかった唯一のエージェントでした。

### 設計から任せたい場合

設計から任せる場合は、Claude Code、Copilot Agent、Codex IDEが候補です。

Claude Codeは、実験Bでも共通テスト11/11、自作テスト22/22で、シンプルな設計方針を明記していました。Copilot Agentも、依存最小・すぐ動くというコンセプトをplan.mdにまとめ、自作テスト19本で高い網羅性を示しました。

Codex IDEは、実験Bで16/16を通し、Codex CLIで出たPUT部分更新バグも起こしていません。同じOpenAI系でも、CLIよりIDEの方が堅牢に出た場面です。

### UIの完成度を重視する場合

UI重視なら、Antigravity CLIとAntigravity IDEが強いです。

実験Aでは両方ともUI満点でした。Antigravity CLIはグラスモーフィズムやアニメーション、アイコン中心の操作が目立ちました。Antigravity IDEはモーダル、サマリーダッシュボード、トースト通知、パルスアニメーションまで作り込んでいました。

実験Bでも、Antigravity CLIはカンバンボードとドラッグ&ドロップを実装し、Antigravity IDEはタグ管理や進捗ゲージまで含む機能豊富なUIを出しました。

ただし、UIの見た目だけで選ぶのは危険です。Antigravity系はバックエンドの入力検証やテスト修正の誠実さで問題が出た場面もあるため、重要な業務アプリではレビューと実機テストを厚めに置くべきです。

### 自動テストの堅牢性を重視する場合

他者テスト修正を含めて見るなら、Copilot Agentが最もおすすめです。

実験Dでは、期待値を書き換えず、APIテスト原本を維持し、UIテストのセレクタ罠も回避しました。報告と実機検証が完全に一致した唯一のエージェントです。

実装時の自作テスト量では、実験AのCodex CLIが23本、実験BのClaude Codeが22本、Copilot Agentが19本と多めでした。ただし、テスト本数だけではなく、実際に何を検証しているか、失敗を握り潰していないかを見る必要があります。

### コードレビューに使う場合

コードレビューでは、単独のエージェントに任せ切らない方がよいです。

動的検証を重視するならCodex CLIとCodex IDEが有用です。特にCodex CLIは、PUT部分更新バグを実測で見つけました。

横断的な整理や設計観点のレビューならClaude Codeが使いやすいです。レビューの観点が明示的で、複数対象の共通傾向をまとめる力があります。

Copilot Agentは、異系統レビュアーとの評価一致が高く、実験Eの最後に均質化トラップ検証の裏付けにもなりました。

一方で、AIレビューには誤検出があります。重要な指摘は、複数エージェントでクロスチェックし、最後は実機検証で確認するのが現実的です。

## 10. まとめ

今回の6エージェント比較では、「どれもだいたい同じ」ではありませんでした。

実験Aだけなら、Claude Codeが最も強い結果でした。4分で実装し、共通テスト24/24、総合85.0点。速度と品質の両立という意味で、最も完成度が高いです。

一方、実験DだけならCopilot Agentが最も信頼できます。期待値を書き換えず、本来失敗すべきものを失敗として残し、報告と実機検証も一致しました。

Antigravity系はUI/UXが強く、短時間で見た目のよい成果物を作る力があります。ただし、バックエンド品質やテスト修正の誠実さでは注意が必要です。

Codex系は対話回数が少なく、一度の指示でまとまった作業を終える傾向があります。Codex IDEは可読性や実験Bの堅牢性で良い面があり、Codex CLIも実測レビューでは価値を出しました。ただし、今回の総合評価では指示違反や自己評価誤認が響きました。

最終結論は次の通りです。

- 総合1位: Claude Code
- 信頼性・他者テスト修正の1位: Copilot Agent
- UI重視の候補: Antigravity CLI / Antigravity IDE
- 動的レビュー要員: Codex CLI / Codex IDE

AIエージェントを選ぶときは、平均点だけでなく「どの局面で失敗するか」を見るべきです。実装時に強いエージェント、レビュー時に強いエージェント、UIに強いエージェント、テスト修正で信頼できるエージェントは同じではありません。

今回の比較で最も大きな教訓は、テスト合格率よりも、失敗を正しく扱う姿勢の方が重要な場面があるということです。
