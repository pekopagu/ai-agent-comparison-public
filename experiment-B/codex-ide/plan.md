# タスク管理Webアプリ 開発プラン

## 1. アプリの目的・ターゲットユーザー

このアプリは、個人または小規模チームが日々のタスクを素早く登録し、状態・優先度・期限で整理できるシンプルなタスク管理Webアプリです。

主なターゲットユーザーは次の通りです。

- 個人の作業・学習・家事などを一覧管理したいユーザー
- 小規模な業務タスクを軽量に管理したいユーザー
- インストール不要のブラウザUIで、ローカル環境でも使えるタスク管理を求めるユーザー

## 2. 機能一覧と優先順位

### 優先度A: 必須機能

- タスク一覧表示
- タスク新規作成
- タスク編集
- タスク削除
- タスク完了・未完了の切り替え
- タスクの優先度設定
- タスクの期限設定
- SQLiteへの永続化
- FastAPIによるREST API
- Vue 3 CDNによる単一ページUI
- pytestによるAPIテスト

### 優先度B: 使いやすさ向上

- 状態別フィルタリング
- 優先度別フィルタリング
- キーワード検索
- 期限順・作成日順の並び替え
- 入力バリデーション
- エラー表示
- 完了数・未完了数などの簡易サマリー

### 優先度C: 今後の拡張候補

- タグ管理
- 複数リスト・プロジェクト管理
- ドラッグアンドドロップ並び替え
- 認証・ユーザー別データ管理
- CSVエクスポート

今回の実装では、優先度AとBを完了範囲とし、CはREADMEに今後の拡張として記載します。

## 3. データモデル設計

SQLiteの `tasks` テーブルを中心に設計します。

| カラム | 型 | 必須 | 説明 |
| --- | --- | --- | --- |
| id | INTEGER PRIMARY KEY AUTOINCREMENT | はい | タスクID |
| title | TEXT | はい | タスク名 |
| description | TEXT | いいえ | 詳細説明 |
| priority | TEXT | はい | `low` / `medium` / `high` |
| due_date | TEXT | いいえ | 期限日。ISO形式 `YYYY-MM-DD` |
| completed | INTEGER | はい | 0: 未完了、1: 完了 |
| created_at | TEXT | はい | 作成日時。ISO形式 |
| updated_at | TEXT | はい | 更新日時。ISO形式 |

APIレスポンスでは `completed` はbooleanとして扱います。

## 4. APIエンドポイント設計

| メソッド | パス | 用途 |
| --- | --- | --- |
| GET | `/api/health` | ヘルスチェック |
| GET | `/api/tasks` | タスク一覧取得。検索・フィルタ・ソート対応 |
| POST | `/api/tasks` | タスク作成 |
| GET | `/api/tasks/{task_id}` | タスク詳細取得 |
| PUT | `/api/tasks/{task_id}` | タスク更新 |
| PATCH | `/api/tasks/{task_id}/toggle` | 完了状態切り替え |
| DELETE | `/api/tasks/{task_id}` | タスク削除 |

`GET /api/tasks` のクエリパラメータは次を想定します。

- `status`: `all` / `active` / `completed`
- `priority`: `all` / `low` / `medium` / `high`
- `q`: タイトル・説明の部分一致検索
- `sort`: `created_desc` / `created_asc` / `due_asc` / `due_desc` / `priority`

## 5. UIデザイン方針（レイアウト・カラー）

業務・個人利用のタスク管理ツールとして、派手なランディングページではなく、起動直後からタスク操作ができる実用画面にします。

### レイアウト

- 上部: アプリ名、簡易サマリー、API状態
- 左または上部の入力エリア: タスク作成・編集フォーム
- メイン: タスク一覧
- 一覧上部: 検索、状態フィルタ、優先度フィルタ、並び替え
- 各タスク行: 完了トグル、タイトル、詳細、優先度、期限、編集・削除ボタン

### カラー

- ベース: 白から淡いグレー
- メインアクセント: 落ち着いたグリーン
- 補助色: ブルーグレー、警告用の赤、優先度用の黄・赤・緑
- 全体として単色に寄りすぎない、読み取りやすい作業用UIにします。

### レスポンシブ対応

- デスクトップではフォームと一覧を横並び
- モバイルでは縦積み
- ボタンや入力欄は折り返し・幅制約を入れて文字がはみ出さないようにします。

## 7. テスト方針

pytestとFastAPIの `TestClient` を使い、APIの振る舞いを中心にテストします。

主なテスト観点は次の通りです。

- `/api/health` が正常応答すること
- タスクを作成できること
- タスク一覧・詳細を取得できること
- タスクを更新できること
- 完了状態を切り替えられること
- タスクを削除できること
- 存在しないIDに404を返すこと
- 不正な優先度などのバリデーションで422を返すこと
- 検索・状態フィルタ・優先度フィルタが機能すること

テストでは本番DBファイルではなく、一時SQLiteファイルを使い、テスト同士が干渉しない構成にします。
