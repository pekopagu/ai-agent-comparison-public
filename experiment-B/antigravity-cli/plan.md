# タスク管理Webアプリ 開発プラン

本ドキュメントは、Python 3.13 + FastAPI + SQLite のバックエンドと、Vue 3 (CDN) を用いたフロントエンドで構築するタスク管理Webアプリの開発計画です。

---

## 1. アプリの目的・ターゲットユーザー

### 目的
個人や少人数チームが日々のタスクを視覚的かつ直感的に管理し、作業の進捗をスムーズに追跡できる環境を提供する。

### ターゲットユーザー
- シンプルで動作が軽く、無駄な機能のないタスク管理ツールを求めている個人
- キーボード操作や直感的なUI（カンバンボード）で効率よくタスクを整理したいエンジニアやクリエイター
- 面倒なセットアップなしで、ブラウザから手軽にタスクを管理したいユーザー

---

## 2. 機能一覧と優先順位

| 優先度 | 機能名 | 詳細 |
| :--- | :--- | :--- |
| **高 (Core)** | タスクのCRUD操作 | タスクの作成、表示、編集、削除機能。 |
| **高 (Core)** | ステータス管理 | 未着手 (Todo)、進行中 (In Progress)、完了 (Done) の3つの状態管理。カンバンボード上でのドラッグ＆ドロップ、またはワンクリックでの移動。 |
| **高 (Core)** | 優先度設定 | 各タスクに「高」「中」「低」の優先度を設定。 |
| **中 (High)** | 期限日管理 | タスクの期日を設定。期限切れタスクを警告色でハイライト。 |
| **中 (High)** | 検索・フィルタ | タイトルや詳細でのキーワード検索、優先度やステータスでのフィルタリング。 |
| **低 (Nice)** | ダークモード | システム設定またはボタンでのライト/ダークテーマの切り替え。 |

## 3. データモデル設計 (SQLite)

### `tasks` テーブル

| カラム名 | 型 | 制約 | 説明 |
| :--- | :--- | :--- | :--- |
| `id` | INTEGER | PRIMARY KEY AUTOINCREMENT | タスクID |
| `title` | TEXT | NOT NULL | タスク名（1〜100文字） |
| `description` | TEXT | | タスクの詳細説明 |
| `due_date` | TEXT | | 期限日 (YYYY-MM-DD) |
| `priority` | TEXT | NOT NULL | 優先度 ('low', 'medium', 'high')、デフォルト 'medium' |
| `status` | TEXT | NOT NULL | ステータス ('todo', 'in_progress', 'done')、デフォルト 'todo' |
| `created_at` | TEXT | NOT NULL | 作成日時 (ISO8601形式: YYYY-MM-DDTHH:MM:SS) |
| `updated_at` | TEXT | NOT NULL | 更新日時 (ISO8601形式: YYYY-MM-DDTHH:MM:SS) |

## 4. APIエンドポイント設計

ベースURL: `/api`

| メソッド | エンドポイント | 説明 | リクエストボディ / クエリパラメータ | レスポンス (JSON) |
| :--- | :--- | :--- | :--- | :--- |
| `GET` | `/tasks` | タスク一覧取得 | クエリ: `status`, `priority`, `q` (検索語) | タスクの配列 |
| `GET` | `/tasks/{id}` | タスク詳細取得 | パスパラメータ: `id` | タスクオブジェクト |
| `POST` | `/tasks` | 新規タスク作成 | `{ title, description, due_date, priority }` | 作成されたタスクオブジェクト |
| `PUT` | `/tasks/{id}` | タスク情報の更新 | `{ title, description, due_date, priority, status }` | 更新されたタスクオブジェクト |
| `DELETE`| `/tasks/{id}` | タスクの削除 | パスパラメータ: `id` | 成功メッセージ `{ "status": "success" }` |

## 5. UIデザイン方針

### レイアウト
- **カンバンボード型レイアウト**:
  - 画面を「未着手 (Todo)」「進行中 (In Progress)」「完了 (Done)」の3つの縦カラムに分割。
  - ヘッダー部に「新規タスク作成」ボタン、検索バー、フィルタオプションを配置。
  - 各タスクはカード形式で表示され、期日、優先度バッジ、編集・削除ボタンを表示。
  - ドラッグ＆ドロップによる直感的なステータス移動（HTML5 Drag and Drop API）をサポート。

### カラー＆タイポグラフィ
- **テーマ**: ダークモードをベースにしたモダンで洗練されたデザイン（Glassmorphismの要素を取り入れる）。
- **主要カラー**:
  - 背景: `slate-900` 風のディープネイビー (`#0f172a`)
  - カード背景: 半透明の `slate-800` (`rgba(30, 41, 59, 0.7)`) + ブラーエフェクト
  - アクセント: ロイヤルバイオレット (`#6366f1`)
- **シグナルカラー**:
  - 優先度「高」/ 期限切れ: ローズレッド (`#f43f5e`)
  - 優先度「中」: アンバーイエロー (`#f59e0b`)
  - 優先度「低」: エメラルドグリーン (`#10b981`)
- **フォント**: `Inter` または `system-ui` を採用し、視認性を重視。

## 6. 開発手順・実装順序

1. **ステップ1: 開発プランの作成と合意 (本ドキュメント)**
2. **ステップ2: バックエンドの開発**
   - データベース接続設定とモデル作成 (`database.py`, `models.py`)
   - FastAPI アプリの初期化と CRUD エンドポイントの実装 (`main.py`, `schemas.py`)
   - SQLite データベースの自動初期化スクリプト作成
3. **ステップ3: pytestによるバックエンドAPIのテスト**
   - テストスイートの作成 (`test_main.py`)
   - APIの正常系・異常系のテストケース実行と確認
4. **ステップ4: フロントエンドの開発**
   - `index.html` (Vue 3 CDN, CSS, JSを1つのディレクトリに構成)
   - `style.css` (Vanilla CSSによるモダンデザイン)
   - `app.js` (Vue 3のロジック、Fetch APIを用いたAPIとの通信)
5. **ステップ5: 動作確認とデバッグ**
   - ローカルでのサーバー起動とブラウザでの動作検証
6. **ステップ6: ドキュメント整理**
   - `README.md` の作成

---

## 7. テスト方針

- **テストツール**: `pytest`, `httpx` (FastAPIのTestClientを使用)
- **テスト用データベース**: 各テスト実行時に一時的なSQLiteファイル、またはインメモリデータベースを作成・破棄し、テストデータが本番環境や他のテストに影響しないようにする。
- **検証項目**:
  - 新規タスク作成のバリデーション（タイトル必須、不正な優先度の拒否など）
  - タスク一覧取得時の検索・フィルタ機能の動作
  - ステータス更新時のデータ整合性
  - 存在しないタスクIDを指定した際のエラーハンドリング (404 Not Found)
