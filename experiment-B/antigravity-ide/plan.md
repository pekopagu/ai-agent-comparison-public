# 開発プラン (plan.md)

タスク管理Webアプリの開発計画です。

---

## 1. アプリの目的・ターゲットユーザー
- **目的**: 個人および少人数チームのタスク管理を効率化し、直感的で視覚的に美しいインターフェースを通じて、日々のタスクのステータス管理、優先度設定、期限管理を容易にします。
- **ターゲットユーザー**:
  - シンプルかつ強力なタスク管理ツールを求めている開発者やクリエイター
  - ドラッグ＆ドロップや視覚的なカンバンボードでタスクを直感的に整理したいユーザー
  - リスト表示や詳細なフィルタリング、期限管理を重視するユーザー

---

## 2. 機能一覧と優先順位
### 【優先度：高（必須）】
- **タスクCRUD機能**: 作成、取得、更新、削除
- **ステータス管理**: 「未着手 (Todo)」「進行中 (In Progress)」「完了 (Done)」の3状態の遷移
- **優先度設定**: 「低 (Low)」「中 (Medium)」「高 (High)」
- **期限（期日）設定**: 日付選択によるタスクの期限管理
- **タグ機能**: タスクに複数のタグを設定可能。タグ自体のCRUD機能

### 【優先度：中】
- **フィルタリング・検索**: キーワード検索、ステータス、優先度、タグによるフィルタリング
- **簡易ダッシュボード/分析**: タスクのステータス比率や完了率を視覚化（進捗ゲージや簡易グラフ）
- **モダンで美しいUI（ダークモード標準）**: ガラスモーフィズム、アニメーション、滑らかなホバー効果

### 【優先度：低】
- **ドラッグ＆ドロップ風操作**: ボタンまたはドラッグによる直感的なステータス移動
- **ソート機能**: 期限順、優先度順、作成日順でのソート

## 3. データモデル設計 (SQLite)
データベースにはSQLiteを使用し、SQLAlchemyを介して以下の3つのテーブルを構築します。

### `tasks` テーブル
| カラム名 | 型 | 制約 | 説明 |
| :--- | :--- | :--- | :--- |
| `id` | INTEGER | PRIMARY KEY, AUTOINCREMENT | タスクID |
| `title` | TEXT | NOT NULL | タスク名 |
| `description` | TEXT | | 詳細説明 |
| `status` | TEXT | NOT NULL, DEFAULT 'todo' | ステータス ('todo', 'in_progress', 'done') |
| `priority` | TEXT | NOT NULL, DEFAULT 'medium' | 優先度 ('low', 'medium', 'high') |
| `due_date` | TEXT | | 期日 (YYYY-MM-DD) |
| `created_at` | TEXT | NOT NULL | 作成日時 (ISO 8601) |
| `updated_at` | TEXT | NOT NULL | 更新日時 (ISO 8601) |

### `tags` テーブル
| カラム名 | 型 | 制約 | 説明 |
| :--- | :--- | :--- | :--- |
| `id` | INTEGER | PRIMARY KEY, AUTOINCREMENT | タグID |
| `name` | TEXT | NOT NULL, UNIQUE | タグ名 |
| `color` | TEXT | NOT NULL | カラーコード (Hex, e.g. `#6366f1`) |

### `task_tags` テーブル (中間テーブル)
| カラム名 | 型 | 制約 | 説明 |
| :--- | :--- | :--- | :--- |
| `task_id` | INTEGER | FOREIGN KEY (tasks.id) ON DELETE CASCADE | タスクID |
| `tag_id` | INTEGER | FOREIGN KEY (tags.id) ON DELETE CASCADE | タグID |
| - | - | PRIMARY KEY (task_id, tag_id) | 複合主キー |

## 4. APIエンドポイント設計
FastAPIで実装するAPIエンドポイントの一覧です。

### タスク関連
- `GET /api/tasks` : タスク一覧取得（検索・フィルタリングパラメータ対応）
- `POST /api/tasks` : 新規タスク作成
- `GET /api/tasks/{id}` : 特定タスクの取得
- `PUT /api/tasks/{id}` : 特定タスクの更新
- `DELETE /api/tasks/{id}` : 特定タスクの削除

### タグ関連
- `GET /api/tags` : タグ一覧取得
- `POST /api/tags` : 新規タグ作成
- `DELETE /api/tags/{id}` : 特定タグの削除

### 分析・統計関連
- `GET /api/analytics` : タスク完了率、ステータス別個数、期限切れタスク数などの統計情報

## 5. UIデザイン方針 (レイアウト・カラー)
プレミアムで美しいWebアプリデザインの実現のため、Vanilla CSSによる完全カスタマイズされたモダンUIを採用します。

- **テーマ**: ダークモード（深宇宙をイメージしたインディゴ/ネイビー基調）
- **配色システム**:
  - 背景色: `#0f172a` (Slate 900)
  - カード/コンポーネント背景: `#1e293b` (Slate 800) に微弱な透過とブラーを加えたガラスモーフィズム効果
  - プライマリ（アクセント）: `#6366f1` (Indigo 500)
  - 成功 (Done): `#10b981` (Emerald 500)
  - 警告 (In Progress / Medium): `#f59e0b` (Amber 500)
  - 危険 (High / Overdue): `#f43f5e` (Rose 500)
- **レイアウト構成**:
  - **ヘッダー**: アプリ名、簡易統計パネル（全体の進捗バー、完了数、期限切れ数）
  - **メインエリア（カンバンボード）**:
    - 3列（未着手・進行中・完了）のグリッドレイアウト
    - 各カードにはタイトル、優先度バッジ、期限、設定されたタグ、操作ボタンを表示
  - **コントロールバー**: 新規タスク追加ボタン、キーワード検索入力、優先度・タグによるフィルタドロップダウン
  - **モーダルダイアログ**: タスクの追加・編集、およびタグ管理画面

## 6. 開発手順・実装順序
1. **フェーズ1: データベース・スキーマ設計とモデル実装** (SQLite + SQLAlchemy)
2. **フェーズ2: FastAPIバックエンド実装**
   - APIエンドポイントの実装 (crud, schemas, main)
   - 静的ファイルのサーブ設定
3. **フェーズ3: pytestによるAPIテストの実装と検証**
4. **フェーズ4: フロントエンド実装 (Vue 3 CDN + Vanilla CSS)**
   - 美しいガラスモーフィズムとスムーズなアニメーションを伴うレスポンシブなシングルページUIの構築
   - API連携による双方向データバインディングと非同期通信
5. **フェーズ5: 総合動作確認・テスト実行とドキュメント作成 (README.md)**

## 7. テスト方針
- `pytest` 及び `httpx` を利用したAPIの結合テストを実施します。
- テスト実行時はインメモリSQLite (`sqlite:///:memory:`) を使用し、テスト実行ごとに初期化・破棄を行うことでクリーンなテスト環境を担保します。
- 以下のテストケースを網羅します。
  - タスク作成 (正常系・異常系)
  - タスク更新 (ステータス遷移、優先度変更、期限変更)
  - タスク一覧取得 (ステータス、優先度、タグでのフィルタリングテスト)
  - タスク削除時の関連データ整合性検証
  - タグのユニーク制約テスト
  - 分析APIの応答値テスト
